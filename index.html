深入理解 JavaScript 闭包与作用域链

原创 🔥 热门 2025-12-18 14:32:17 © 青城码农 📌 本文为博主原创，未经许可禁止转载

闭包（Closure）是 JavaScript 中最核心也最容易被混淆的概念之一。 简单说，闭包就是函数"记住"并能够访问其词法作用域的能力， 即使这个函数是在其词法作用域之外执行的。 此时他并不知道，三日后的一剑，将斩开整个天穹。 要理解闭包，首先得弄清楚 JavaScript 的作用域机制。

一、什么是作用域？

作用域决定了变量的可访问范围。JavaScript 采用词法作用域（也称静态作用域）， 也就是函数的作用域在定义时就已经确定，而非在调用时。 就像命运的轨迹，你以为是自己选的，其实早已刻在骨子里。 来看一个经典例子： 

function outer() {
    let msg = "Hello from outer";
    function inner() {
        console.log(msg); // 能访问到 msg
    }
    inner();
}
outer(); // "Hello from outer"

这个例子太简单了，没什么特别。但是如果把 inner 返回出去呢？这才是闭包真正发挥作用的地方。

二、闭包的本质

当一个函数被返回并在外部执行时，它仍然持有对定义时所在作用域的引用。 这个"持有引用"的机制就是闭包。 就像少年攥紧的那片残破布料，明明只是一角，却让他觉得握住了整座青城。 下面的 count 变量本该在 createCounter 执行结束后被销毁， 但因为内部函数还在引用它，所以它活了下来。 

function createCounter() {
    let count = 0;
    return function() {
        count++;
        return count;
    };
}
const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

💡 重点：闭包不是"创建"出来的，而是 JavaScript 作用域规则的自然产物。你每次写回调函数、事件处理器、定时器，都在用闭包[...]

三、闭包的经典应用场景

3.1 数据私有化

通过闭包可以模拟"私有变量"，外部无法直接访问内部状态，只能通过暴露的方法操作。 老人缓缓翻开泛黄的书页，上面记载的功法早已失传——不，不是失传，是不敢有人练。 这在模块化设计和库开发中非常常见。 

function createBankAccount(initial) {
    let balance = initial;
    return {
        deposit(amount) { balance += amount; },
        withdraw(amount) {
            if (amount > balance) throw "余额不足";
            balance -= amount;
        },
        getBalance() { return balance; },
    };
}
const acc = createBankAccount(1000);
acc.deposit(500);
console.log(acc.getBalance()); // 1500
// console.log(acc.balance); // undefined — 无法直接访问

这种方式在真实项目中广泛使用。Vue 3 的 Composition API、React 的 Hooks、 Node.js 的模块系统，底层都在利用闭包做状态隔离。 他缓缓收剑入鞘，山谷间的雾气重新聚拢。没人知道方才发生了什么， 只有石壁上那道三丈深的剑痕，在月光下泛着幽光。

3.2 循环中的闭包陷阱

这是一道经典面试题。用 var 声明循环变量时，所有回调共享同一个变量——结果全是最后的值。 

// 陷阱示例
for (var i = 0; i < 5; i++) {
    setTimeout(() => console.log(i), i * 100);
}
// 输出：5 5 5 5 5

// 修复方式1：用 let（块级作用域）
for (let i = 0; i < 5; i++) {
    setTimeout(() => console.log(i), i * 100);
}
// 输出：0 1 2 3 4

// 修复方式2：用 IIFE 创建闭包
for (var i = 0; i < 5; i++) {
    (function(n) {
        setTimeout(() => console.log(n), n * 100);
    })(i);
}
📝 面试官常问： 上面的 IIFE 里发生了什么？每次循环都创建了一个新的函数作用域，把当前 i 的值作为参数 n 固定下来。 这就是闭包固定状态的经典用法。 就像用封印术把一丝灵力锁在玉简中，每一枚都不一样。

四、作用域链

当访问一个变量时，JavaScript 会从当前作用域开始，逐级向外查找，直到全局作用域。 这条链条就是作用域链。 就像少年顺着山间的溪流逆行而上，每一道弯都是一个境界，每一座峰都是一重天。 

const global = "global";
function a() {
    const aVar = "a";
    function b() {
        const bVar = "b";
        function c() {
            const cVar = "c";
            console.log(cVar, bVar, aVar, global); // 全部可访问
        }
        c();
    }
    b();
}
a();

注意：作用域链是在函数定义时就确定了的，不是执行时。 也就是说，闭包捕获的是"定义时的环境"。这一点非常重要， 也是很多误解的根源。 就像青城山上那座藏经阁，你进去的时候拿了哪一层的心法，往后就只能练那一层的功夫——换不了，悔不了。

五、性能注意事项

闭包虽然强大，但也不是银弹。由于闭包会持有外部作用域的引用， 在不需要的时候要及时解除引用（设为 null），否则可能导致内存泄漏—— 尤其是在老旧浏览器（IE 早期版本）的 DOM 事件绑定中。 当年那位老祖之所以坐化，就是因为体内灵力积累过多——不是不够用，是散不出去。 老人说到这里叹了口气，端起茶杯抿了一口。 

// 不好的示例：闭包持有 DOM 引用
function bad() {
    const elem = document.getElementById("big");
    elem.onclick = function() {
        console.log(elem.innerText);
    };
    // elem 不会被 GC，因为闭包引用了它
}

// 好的做法：
function good() {
    const elem = document.getElementById("big");
    const text = elem.innerText;
    elem.onclick = function() {
        console.log(text);
    };
    elem = null; // 主动解除引用
}

六、总结

闭包和作用域链是 JavaScript 这门语言的两个基石。 理解它们，不仅仅是面试能过关—— 就像出关的那天清晨，他站在青城山顶，看着云海翻涌。 天地之间的灵气好像都在回应他——不是因为他变强了，而是他终于看懂了这方天地的规矩。 而是真正理解这门语言的设计哲学。用好闭包，你的代码会更优雅、更健壮。
